# Contributing to the ALIGN framework

We welcome contributions that improve the clarity, accuracy, or usefulness of this framework. Whether you spotted a typo, have a better way to phrase a concept, or want to propose a new template, here is how to get involved.

---

## How to contribute

1. **Fork** this repository.
2. **Create a branch** with a descriptive name (e.g. `fix/scorecard-typo` or `feature/industry-playbook-saas`).
3. **Make your changes.** Keep edits focused — one improvement per pull request makes review faster.
4. **Test your links.** If you added or renamed any markdown files, verify that all internal links still resolve correctly.
5. **Open a pull request** against `main` with a short description of what you changed and why.

---

## What makes a good contribution

- **Clarity over cleverness.** The framework should read like a well-organized consulting document, not a marketing page.
- **Proper punctuation and sentence case.** Avoid unnecessary capitalization. Finish sentences with a period.
- **No marketing fluff.** Replace words like "powerful," "revolutionary," or "game-changing" with precise, descriptive language.
- **Consistency.** Match the tone, structure, and formatting of existing documents.

---

## Reporting issues

If you find a broken link, an inconsistency between documents, or a factual error, please open an issue describing:

- Which file is affected.
- What the problem is.
- What you think the correct version should be (if applicable).

---

## Code of conduct

Be respectful, constructive, and professional. We are building a practical resource for strategists and marketers — contributions should reflect that standard.

---

## Questions

If you have questions about the framework itself or want to discuss a larger contribution, feel free to open an issue or reach out to kevin@dishine.it.
